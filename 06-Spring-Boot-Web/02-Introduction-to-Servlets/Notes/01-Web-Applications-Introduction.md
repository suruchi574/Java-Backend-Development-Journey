# Introduction to Web Applications

A **Web Application** is a software application that runs on a **web server** and is accessed through a **web browser** or **mobile application** over the internet or a local network.

Examples:
- Gmail
- Facebook
- Amazon
- Flipkart
- Instagram

---

# Static Content

Static content refers to web pages whose content remains the same for every user unless a developer manually updates the files.

### Characteristics

- Fixed content
- Faster to load
- Does not interact with databases
- Same response for every user

### Technologies Used

- HTML
- CSS
- JavaScript (Static Files)
- Images

### Examples

- Portfolio website
- Company information page
- Contact Us page

---

# Dynamic Content

Dynamic content is generated in real time based on user requests, server-side processing, or data stored in a database.

The content can be different for different users.

### Characteristics

- Content changes dynamically
- Interacts with databases
- User-specific responses
- Generated by server-side code

### Examples

- Gmail Inbox
- Online Banking
- Shopping Cart
- Social Media Feed
- Flight Booking System

---

# Data Handling in Web Applications

A web application follows the **Client-Server Architecture**.

### Client

The client can be:
- Web Browser
- Mobile Application
- Desktop Application

The client sends a request to the server.

### Server

The server:
- Receives the request
- Processes the request
- Interacts with the database (if required)
- Sends a response back to the client

---

# Client-Server Communication

Nowadays, the client and server exchange data mostly in **JSON (JavaScript Object Notation)** format.

### Client → Server

The client sends data to the server in JSON format.

Example:

```json
{
  "name": "Suruchi",
  "city": "Bangalore"
}
```

### Server → Client

The server processes the request and returns a JSON response.

Example:

```json
{
  "message": "Data received successfully"
}
```

JSON is lightweight, easy to read, and supported by almost every programming language.

---

# Server-Side Processing

Server-side processing refers to all operations performed on the server before sending the response to the client.

These operations include:

- Processing user requests
- Executing business logic
- Validating data
- Querying databases
- Creating objects
- Generating dynamic content
- Sending responses

Java frameworks like **Spring** and **Spring Boot** simplify server-side processing.

---

# What is a Servlet?

A **Servlet** is a Java server-side component that receives client requests, processes them, and sends responses back to the client.

It acts as a bridge between the client and the server.

### Responsibilities of a Servlet

- Accept client requests
- Read request data
- Perform business logic
- Interact with the database (if required)
- Generate responses
- Send responses back to the client

---

# Why Can't We Run a Servlet Like a Normal Java Program?

A normal Java application starts execution from the **main()** method and runs directly on the **JVM**.

Example:

```java
public static void main(String[] args)
```

A Servlet does **not** have a `main()` method.

Instead, it is managed by a **Servlet Container**, which creates the servlet object, manages its lifecycle, and invokes its methods whenever a client sends a request.

Therefore, a Servlet cannot run directly on the JVM like a console application.

---

# Servlet Container (Web Container)

A **Servlet Container** (also called a **Web Container**) is a runtime environment that manages Java Servlets.

It loads servlets, handles HTTP requests, and manages the complete lifecycle of a servlet.

### Responsibilities

- Loads Servlet classes
- Creates Servlet objects
- Manages the Servlet lifecycle
- Handles HTTP requests and responses
- Performs URL mapping
- Provides session management
- Manages security

---

# Apache Tomcat

**Apache Tomcat** is one of the most popular lightweight Servlet Containers and Web Servers used to run Java web applications.

### Features

- Open Source
- Lightweight
- Easy to configure
- Supports Servlets and JSP
- Widely used in Java web development

---

# Installing Apache Tomcat

### Step 1

Download Apache Tomcat from the official website.

Example Version:

```text
Apache Tomcat 10.1.16
```

### Step 2

Extract (Unzip) the downloaded ZIP file.

### Step 3

Important Folder

```text
webapps
```

The **webapps** folder is where deployed web applications (WAR files) are placed.

### Step 4

Start Tomcat

Windows

```text
bin/startup.bat
```

Linux/macOS

```text
bin/startup.sh
```

### Step 5

Stop Tomcat

Windows

```text
bin/shutdown.bat
```

Linux/macOS

```text
bin/shutdown.sh
```

---

# Embedded Tomcat

Instead of installing Tomcat separately, modern Java applications often use **Embedded Tomcat**.

Embedded Tomcat is included as a dependency inside the project and starts automatically when the application runs.

### Advantages

- No separate installation
- No manual deployment
- Easy development
- Portable
- Used by Spring Boot

---

# JAR vs WAR

Java applications can be packaged into different archive formats.

## JAR (Java Archive)

Used for **Console/Desktop Applications**.

Extension:

```text
.jar
```

Runs directly on the JVM.

---

## WAR (Web Archive)

Used for **Web Applications**.

Extension:

```text
.war
```

Runs inside a Servlet Container such as Apache Tomcat.

---

# Creating an Embedded Servlet Project

## Step 1

Create a new **Maven Project**.

---

## Step 2

Enter the following project details:

| Property | Value |
|----------|-------|
| JDK | 17 |
| Archetype | `maven-archetype-quickstart` |
| Archetype Version | 1.5 (or the version used in the course) |
| Project Name | ServletEx |
| Group Id | com.barbighaiya |
| Version | 1.0-SNAPSHOT |

---

## Step 3

Since the Quickstart project is a normal Java project, add the required dependencies in the `pom.xml` file.

### Required Dependencies

1. Jakarta Servlet API
2. Apache Tomcat Embedded Core

---

## Step 4

Open **Maven Repository** and search for:

### Jakarta Servlet API

Copy the dependency and paste it into `pom.xml`.

### Tomcat Embedded Core

Choose a stable version (preferably the one used in the course), copy the dependency, and paste it into `pom.xml`.

---

## Step 5

Update the Maven Project

After adding the dependencies:

- Right-click the project.
- Select **Maven → Update Project**.
- Click **OK**.

Maven automatically downloads all required libraries.

---

# Summary

- A **Web Application** runs on a web server and is accessed through a browser or mobile app.
- **Static Content** remains the same unless manually updated.
- **Dynamic Content** is generated at runtime based on user requests or database data.
- Clients and servers typically exchange data using **JSON**.
- **Server-side processing** includes business logic, validation, and database operations.
- A **Servlet** is a Java server-side component that processes client requests and generates responses.
- Servlets require a **Servlet Container (Web Container)** because they cannot run directly on the JVM.
- **Apache Tomcat** is a lightweight Servlet Container used to run Java web applications.
- Java console applications are packaged as **JAR** files, while web applications are packaged as **WAR** files.
- Modern applications commonly use **Embedded Tomcat**, eliminating the need for manual Tomcat installation and deployment.
- To create an Embedded Servlet project, create a Maven project, add the **Jakarta Servlet API** and **Tomcat Embedded Core** dependencies, and update the Maven project.
